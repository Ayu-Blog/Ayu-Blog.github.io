    学习计划

    ​ 博主已经有一定的 C 语言基础，但之前在项目中一直感觉 C 语言学习不够扎实，因此打算从头再整理一遍，打好语言基础更有利于自己今后开发。

    也由于自己有一定的基础，学习过程中以概念知识为辅，实践为主。

    C语言简介

    ​ C 可能是最为人知的编程语言，它被全世界的计算机科学课程中用作参考语言。

    ​ C 语言也是我接触的第一门语言，在大学期间 C 语言被我们用来学习编程，但是 C 语言并不是一门学术型语言，C 是一门非常底层的编程语言。

    ​ C 有着鲜明的特点，如下：

    高级语言

    ​ 有意思的事，在百度搜索的时候能看见这一特点有写高级语言、中级语音、低级语言。然后我就去了解一下。

    低级语言的分类有两种：

    1.机器语音

    ​ 机器能够识别状态为”断开”和“闭合”两个状态，也就是 0 和 1。

    ​ 因此使用机器语言需要记忆全套的机器码，不仅开发过程繁琐、消耗时间长，代码的可读性也非常低。

    ​ 为了让软件工程师更高效的编写程序，汇编语言（assembly language）被发明出来。

    2.汇编语音

    ​ 汇编语言采用人类容易记忆和识别的助记符来代表一些0和1的指令，比如AND代表加法。

    ​ 汇编语言的代码不能被计算机直接执行，需要通过汇编程序将它们转换成可执行的机器代码。虽然机器代码已经被容易记忆的英文符号代替，但是程序的开发难度依然很高。程序开发时不但需要控制硬件计算器和寄存器等，还需要解决数据在硬件中存储的地址转换等问题。

    中级语言的特点：

    1. 语法相对简单：中级编程语言的语法通常比低级编程语言更简洁，更易于理解和学习。这使得开发人员可以更快地上手并编写高质量的代码。
    2. 抽象程度适中：中级编程语言提供了一定的抽象，使得开发人员能够更好地组织和管理代码。它们通常提供了数据结构、函数和模块等高级特性，使得代码更易读、可维护和可扩展。
    3. 性能优化能力：与高级编程语言相比，中级编程语言通常更接近底层硬件，可以更好地控制和优化代码的性能。开发人员可以使用底层的数据结构和算法，以及直接操作内存和硬件资源的能力，来提高程序的执行效率。

    高级语言

    ​ 我们现在大多数人使用的语言，如C、C++、Python、Java、Matlab、LabVIEW等等，都属于高级语言，相对于低级语言，它更接近于我们平时正常的人思维，其最大的特点是编写容易，代码可读性好。实现同样的功能，使用高级语言耗时更少，程序代码量更短，更容易阅读。其次，高级语言是可移植的，也就是说，仅需稍作修改甚至不用修改，就可将一段代码运行在不同类型的计算机上。

    ​ 其缺点也很明显，使用高级语言编写的程序运行时，需要先将其翻译成低纸语言计算机才能运行它，在翻译过程中可能程会产生一些多余的部分，运行效率低些。另外，对硬件的可控性相对于低级语言弱些，目标代码量较大。

    ​ 综上

    ​ 低级语言更接近于机器，高级语言更接近于人类。C 语言是一门非常底层语言，于机器硬件、寄存器地址、内存打交道，但是语言语法上更接近于人类理解，因此 C 语言是一门高级语言。

    编译型语言

    ​ C 语言是一门编译型语言。再次我们需要了解语言类型有：编译型语言和解释型语言。不同的语言有着不同的规定:

    ​ 编译型语言：

    ​ “一次编译，无限执行”：只要生成了可执行程序，就能随时执行。当运行的时候，只需要编译可执行代码就可，不再需要编译器以及源代码。

    ​ “可执行程序以及源程序不可跨平台”：不同操作系统对可执行文件的内部结构有着不一样的要求，彼此之间不兼容。相同的操作系统也不一定兼容，例如X64程序拿到X86上依然不能执行。

    ​ 不同平台对于函数、变量等定义也不一样，因此源代码也无法跨平台执行。

    ​ 解释型语言：

    ​ 每次执行程序都需要转换一次源代码，因为每次执行的功能可能不一样，所以每次转换的代码也会有所不同。所以解释型语言执行效率天生低于编译型语言。
