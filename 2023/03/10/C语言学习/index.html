​ 边学习边记录，方便自己以后回顾总结

初识C语言

C语言的优点

  • 高效性

    C语言上高效的语言，充分利用当前计算机的优势，运行速度很快。

    具体理解：原因是C语言和汇编指令几乎能直接翻译，而机器语言和汇编又能直接翻译，所以在写C语言的时候，代码的效率几乎和机器语言效率一样，但是这也取决于具体使用的编译器，不同编译器对代码的翻译不一样，例如，有的编译器会理解代码无语语句并删除。C语言开发之处就是为了补汇编语言不能跨平台和模块化不好等…，C语言在开发操作系统是非常适用的，因为C语言有指针，C语言可以直接操作内存地址，使得我们不用使用汇编，但是汇编也是少不了的，不过只有很少的一部分我们需要用汇编，因为C语言没有直接的语句与之对应。比如LGDT等指令。

  • 可移植性

    C语言是可移植的语言，这表示在系统中编写的C程序稍作修改或不修改就能在其他系统运行，如需修改也只需修改主程序头文件中少许项即可。

    具体理解：C语言标准、代码库和编译器

    • 标准：C语言的标准由ISO(国际标准化组织)和ANSI(美国国家标准化组织)共同制定，在全球范围内得到广泛应用，因此在不同的平台上编写的C语言程序可以被编译器和操作系统所支持。
    • 标准库：C语言提供了标准库函数，这些函数在不同的平台上具有相同的语义和行为，这使得C语言程序可以在不同平台上进行编译和执行。
    • 编译器：C语言的编译器通常可以针对不同的处理器架构生成不同的目标代码，这样就可以在不同的处理器架构中使用相同的代码。

  • 强大而灵活

    • 底层语言：它允许程序员直接访问计算机的底层硬件资源，如内存和处理器，这使得C语言能够高效地处理大量数据和执行复杂的计算任务。
    • 数据类型：C语言提供了丰富的数据类型，包括基本数据类型（如整型、浮点型、字符型等）和复合数据类型（如数组、结构体、联合体等），这使得C语言可以处理各种类型的数据和复杂的数据结构。
    • 库函数丰富：C语言内置了很多库函数，例如stdio.h和stdlib.h等，这些库函数提供了很多常用的功能，例如输入输出、内存分配和数学运算等。此外，程序员也可以编写自己的库函数，以扩展C语言的功能。
    • 强大的指针：C语言提供了指针，它是一个非常强大和灵活的概念。使用指针，程序员可以直接访问内存地址和修改内存内容，这使得C语言能够实现很多高级的数据结构和算法。
    • 综上所述，C语言之所以被认为是强大而灵活的，是因为它允许程序员直接访问底层硬件资源、提供了丰富的数据类型、库函数和指针等强大的功能，并且具有可移植性。

  • 面向程序员

    C语言是为了满足程序员的需求而设计的，程序员利用C可以访问硬件、操控内存中的位。C语言由丰富的运算符，能让程序员简洁地表达自己的意图。大多数C实现都有一个的大型的库，包含众多的有用的C函数。这些函数用于处理程序员经常需要解决的问题。

语言标准

第一个ANSI/ISO C标准

​ 什么是ANSI C、ISO C、C89、C90标准？

​ 随着C语言使用得越来越广泛，C 社区意识到需要一个更全面、更新颖、更严格的标准。鉴于此，美国国家标准协会（ANSI）与1983年组建了一个委员会（X3J11），开发了一套新标准，并于1989年正式公布，该标准（ANSI C）定义了C语言和C标准库。国际标准化组织与1990年采用了这套C标准（ISO C）。因此ISO C和ANSI C是完全相同的标准。ANSI/ISO标准的最终版本通常叫做C89（ANSI于1989年批准该标准）/ C90（ISO于1990年批准该标准）。

​ 该委员会制定的指导原则中，最有趣的可能是：保持C的精神。委员会在表述这一精神时列出了一下几点：

  • 信任程序员；
  • 不要妨碍程序员做需要做的事；
  • 保持语言精练简单；
  • 只提供一种方法执行一项操作；
  • 让程序运行更快，即使不能保证其可移植性；

C99标准

​ 1994年，ANSI/ISO联合委员会（C9X委员会）开始修订C标准，最终发布C99标准。

​ 该委员会遵循了最初C90标准原则，包括保持语言的精练简单。委员会用意不是在C语言中添加新特性，而是为了达到新目标。

  • 支持国际化编程，例如提供多种方法处理国际字符集；
  • 调整现有实践致力于解决明显的缺陷；
  • 为适应科学和工程中关键数值计算，提高C的适应性；

C11标准

​ 标准委员会咋2007年承诺C标准下一个版本是C1x，2011年终于发布了C11标准。

​ 此次，委员会提出了一些新的指导原则，处于对当前编程安全的担忧，不那么强调”信任程序员”目标了。并且，供应商并未像对C90那样很好接收和支持C90。这表示C99一些特性成为C11的可选项，同时需要强调的是，修订标准的原因不是因为原标准不能用，而是需要跟进新的技术。

使用C语言的7个步骤

  1. 第一步：定义程序的目标

​ 在动手写代码之前，要在脑中有清晰的思路，想要程序去做什么首先自己需要明确自己想要做什么，思考程序需要哪些信息，要进行哪些计算和控制，以及程序应该要报告什么信息，这一步中，不需要具体的计算机语言，应该用一般术语描述问题。

  1. 第二部：设计程序

​ 对程序需要完成什么任务有概念性的认识后，就应该考虑如何用程序完成它。例如，用户界面是怎么样的？如何组织程序？目标用户是谁？准备花多长的时间来完成这个程序？

​ 除此之外，还要决定在程序中如何表示数据，以及用什么方法处理数据，学习语言之初，遇到的问题都很简单，没什么可选的，但是，随着要处理的情况越来越复杂，需要决策和考虑的方面也越来越多，通常，选择一个合适的方式表示信息更容易设计程序和处理程序。

  1. 第三步：编写代码

​ 设计好程序后，就可以编写代码来实现它，也就是说，把你设计的程序翻译成C语言。这里就是真正需要使用C语言的地方。可以把思路写在纸上，但是最终还是要把代码输入到计算机中。

  1. 第四步：编译

​ 这一步为编译源代码，编译的细节取决于编程环境。

​ 编译器是把源代码转换成可执行代码的程序，可执行代码是用计算机的机器语言表示的代码。这种语言由数字码表示的指令组成。

  1. 第五步：运行程序

​ 传统上，可执行文件是可运行的程序，在常见环境中中运行程序要输入可执行文件的文件名，而其他环境运行命令用也可以选择菜单中的选项或按下特殊键来编辑和执行C程序，最终的程序可通过单机或双击文件或图标直接在操作系统上运行。

  1. 第六步：测试和调试程序

​ 程序能运行是个好迹象，但有时也会出现运行错误。因此，需要检查程序是否按照你所设计的思路运行，你会发现你的程序可能存在一些错误，计算机行话叫做BUG。查找并修复程序错误的过程叫调试。学习的过程中不可避免的会犯错，学习编程亦是如此；

  1. 第七步：维护和修改代码

​ 创建程序，你发现程序有错，或者扩展程序的用途，这就需要修改程序。如果在编写程序的时候清楚地做了注释并采用合理的设计方案，这一步就会轻松很多。

  1. 说明

​ 编程并不是描述的七步这样是一个线性的过程。有时，需要我们在不同的步骤之间往复。

​ 许多初学者经常忽略第一步和第二步，直接跳到第三步。刚开始学习时，编写的程序非常简单，完全可以在脑中构思整个过程。即使写错了，也很容易发现。但是随着程序越来越庞大，越来越复杂，动脑不动手可不行，而且程序中隐藏的错误也越拉越难找，最终跳过前两个步骤的人往往浪费更多的时间。

编程机制

​ C语言的编程Microsoft Office未激活机制：通过程序把源代码文件转换为可执行文件（其中包含可直接运行的机器语言代码）。典型C实现通过编译和链接两个步骤来完成这一过程。编译器把源代码转换成中间代码（如：机器语言代码），链接器把中间代码和其他代码合并，生成可执行文件。另外，链接器还将编写的程序和预编译的库代码合并。

​ 将源代码转换为中间代码并放在目标代码文件（简称：目标文件），目标文件并不是一个完整的程序。目标代码文件缺失启动代码。启动代码相当于程序与操作系统间的接口。同时目标代码文件还缺少库函数，几乎所有的库函数都需要使用C库（C标准库）中的函数。

​ 链接器的作用：把编写的目标代码、系统的标准启动代码和库代码这三部分合并成一个文件（可执行文件）。

第一章学习总结

  • C是强大而简洁的编程语言。目前有大量的学习资料，大量的实用编程工具，可以和好控制硬件，是具有底层语言的高级语言；
  • C是编译型语言。C编译器和链接器是把C语言源代码转换成可执行代码的程序。
  • 用C语言编程可能费力、困难，让你感到沮丧，但是它也可以激发你的兴趣，让你兴奋、满意。

第二遍：程序细节

#include指令和头文件

​ 这是程序的第一行。相当于是把 stdio.h 文件中的所有内容都输入该行的位置。实际上，这是一种 “拷贝-粘贴” 的操作。

​ #include 这行代码是一条 C 预处理指令。通常，C 编译器会在编译前对源代码做一些准备工作，即预处理。

​ 所有的 C 编译器软件包都提供 stdio.h 文件。该文件中包含了供编译器使用的输入和输出函数（如printf（））信息。该文件的含义是标准输入/输出头文件。

main()函数

​ C 程序一定从 main() 函数开始执行（目前不考虑例外的情况）。除了 main() 函数，我们可以任意命名其他函数，而且 main 函数必须是开始的函数。圆括号的作用就是识别 main 是一个函数，后续还会有更多函数学习，函数是 C 语言的基本模块。

​ int 是 main 函数的返回类型。这表明 main 函数返回的值是整数，返回给操作系统。

​ 通常情况，圆括号中包含一些传入参数信息，实例中没有任何传入参数，因此圆括号内的单词是 void。

注释

​ 在程序中，被 /* */ 两个符号括起来的部分就是程序的注释。写注释能让他人（包括自己）更容易明白你所写的程序。C 语言注释的好处之一是，可将注释放在任意的地方，甚至是与要解释的内容的同一行。较长的注释可单独放一行或多行。

花括号、函数体和块

​ 在实例程序中花括号把 main() 函数括起来。一般而言，所以的 C 函数都使用花括号标记函数体的开始与结束。这是规定，不能省略。只有花括号（{}）能起到这种作用，其他形式都不行。

声明

​ 示例中，这行代码叫做声明。声明是 C 语言中最重要的特性之一。

​ 在示例中声明完成了两件事，其一，在函数有一个名为 num 的变量。其二，int 表明 num 变量是一个整数。int 是一个数据类型。编译器使用这些信息为 num 变量在内存中分配存储空间。

​ int 是 C 语言中的一个关键词，表示一种基本的 C 语言数据类型。关键词是语言定义的单词，不能用作其他的用途。例如，用 int 作为函数名和变量名。

​ num 是一个标识符，也就是一个变量、函数或其他实体的名称。因此，声明把特定标识符与计算机内存中的特定位置联系起来，同时确定了储存在某位置的信息类型或数据类型。

​ 在C语言中，所有的变量都必须先声明才能使用。这意味着必须列出程序中用到的所有变量名以及其类型。

数据类型

​ C 语言可以处理多中数据的类型，如整数、字符和浮点数。把变量声明为整型或字符类型，计算机才能正确地储存、读取和解释数据。

命名

​ 给变量命名是要使用有意义的变量名或标识符。如果变量名无法清除地表达自身的用途，可以注释中进一步说明。这是一种良好的编程习惯和编程技巧。

​ C99 和 C11 允许使用更长的表示名，但是编译器只识别前 63 个字符。

​ 可以用小写字母、大写字母、数字和下划线；来命名。并且，名称的第一个字符必须是字符或下划线，不能是数字。

​ 操作系统和 C 库经常使用以一个或两个下划线字符开始的标识符，因此最好避免在自己的程序中使用这种名称。标准标签都以一个或两个下划线字符开始。这样的标识符都是保留的。这表示，虽然我们使用没有语法错误，但是会导致名称冲突。

​ C 语言的名称区分大小写。

return语句

​ return 语句是示例代码的最后一条语句。int main（void）中的 int 表面 main() 函数应返回一个整数。C 标准要求 main() 这样做。有返回值的 C 函数要有 return 语句。如果遗漏 main() 函数中 return 语句，程序在运行至最外面的右花括号（}）时会返回 0.。因此，可以省略 main() 函数末尾止的 return，但是不要在其他有返回值的函数中漏掉它。因此，强烈建议养成 main() 函数中保留 return 语句的好习惯。
