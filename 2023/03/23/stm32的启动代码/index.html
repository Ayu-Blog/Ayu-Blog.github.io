STM32 启动代码学习

边看启动文件边学汇编

汇编基础知识链接

ARM汇编指令集：https://blog.csdn.net/zlmm741/article/details/105189487

指令 作用
EQU 取符号名（类似C #define)，同义词 *
AREA 指示编译器汇编一个新段（代码段或数据段）
SPACE 分配内存空间并填零。[标号] SPACE [表达式]， 同义词 %
PRESERVE8 按8字节对齐
EXPORT 声明全局，可被外部文件使用，同义词 GLOBAL
DCD 以字为单位分配内存，要求4字节对齐且初始化该内存
PROC 定义子程序，与ENDP成对使用，表示子程序结束 同义词 FUNCTION
WEAK 编译器特性。弱定义，优先使用外部文件定义的标号。
IMPORT 声明标号来自外部文件，类似于C extern
B 跳转到一个标号
ALIGN 编译器指令，对指令或数据存放地址进行对齐（一般跟一个立即数，缺省为4字节）
END EOF，文件结束
IF,ELSE,ENDIF 条件分支

启动文件详解

栈

​ 注意，汇编里的注释符是;

​ 也就是 通知编译器链接 -> 分配一片栈空间 -> 记录栈顶地址

​ 栈的作用是用于局部变量，函数调用，函数形参等的开销，栈的大小不能超过内部 SRAM 的大小。如果编写的程序比较大，定义的局部变量很多，那么就需要修改栈的大小。如果某一天，你写的程序出现了莫名奇怪的错误，并进入了硬 fault 的时候，这时你就要考虑下是不是栈不够大，溢出了。

汇编指令详解

EQU

指令讲解如下：

  • EQU

​ EQU 指令为数值常量、寄存器相关值或 PC 相关值提供符号名称。

  • Syntax

​ EQU 名称表达式 {, 类型}

  • name(名称):是分配给值的符号名称。

  • expr(表达式):是寄存器相对地址、PC 相对地址、绝对地址或 32 位整型常量。

  • typr(类型):是可选的。可以是任何一个:

    • ARM
    • THUMB
    • CODE32
    • CODE16
    • DATA

  • 您只有在表达式内量为绝对地址的时候才能使用。如果导出，则根据 ，目标文件中符号表中的条目将标记为 ARM、THUMB、CODE32、CODE16 或 DATA。这可以由链接器使用。

  • Usage

​ 使用 EQU 定义常量。这类似于在 C 中使用#define 定义常量。

  • Examples

​ EQU 是伪指令，不生成二进制代码。另外，Label 就是一个标号，加不加只是为了规范，不影响二进制结果。

SPACE

指令讲解如下：

  • SPACE or FILL

​ SPACE 指令保留一个归零的内存块。 FILL 指令保留一块内存以填充给定值。

  • Syntax
  1. {label} space expr
  2. {label} FILL expr{ ，value{，valuesize}}
  • label

​ 任意的一个标签

  • expr

​ 给定要填充的字节数或零。

  • value

​ 计算结果为用于填充保留字节的值。是可选的，如果省略，则为 0。在 NOINIT 区域中必须为 0。

  • valuesize

​ size的大小，单位是字节，可选可忽略

  • Usage

​ 使用 ALIGN 指令对齐 SPACE 或 FILL 指令之后的任何代码。

  • Example

AREA

​ 伪指令指示汇编程序汇编新的代码或数据段。

  • Syntax

​ AREA sectionname{，attr}{，attr}…

  • sectionname

​ 是要为该部分指定的名称。Section（段）是由链接器操作的独立、命名、不可分割的代码或数据块。

​ 您可以为分区选择任何名称。但是，以非字母字符开头的名称必须括在条形中，否则将生成缺少节名称错误。例如|1_data|。

​ 需要注意的是某些名称不能自己用的。例如，|.text|是用于 C 编译器生成的代码Section（段），或与 C 库关联的代码Section（段）。

  • attr

​ 是一个或多个逗号分隔的部分属性。有效属性为：

  1. ALIGN = expression (对齐为表达式)

​ 默认情况下，ELF 部分在四字节边界上对齐。可以是 0 到 31 之间的任何整数值。该部分在 2^expression 字节边界上对齐。例如，如果表达式为 10，则该部分在 1KB(2^10) 边界上对齐。

​ 这与指定 ALIGN 指令的方式不同。

  1. ASSOC = section

​ 指定关联的 ELF 部分。必须包含在包含的任何链接中 section sectionname section

  1. CODE

​ 在机器语言中。只读是默认值。

  1. CODEALIGN

​ 当在节内的 ARM 或 Thumb 指令之后使用 ALIGN 指令时，导致 armasm 插入 NOP 指令，除非 ALIGN 指令指定不同的填充。 CODEALIGN 是只执行部分的默认值。

后略

了解一定汇编后再看栈

堆

​ 堆用于 malloc/calloc 等用于动态分配内存的函数。

​ 通知编译器链接 -> 记录堆头地址 -> 分配一片堆空间 -> 记录堆尾地址

​ 静态变量和局部变量都是在 SRAM 中分配

过渡

向量表

EXPORT/GLOBAL声明全局，可被外部文件使用

作用

​ 当内核响应了一个发生的异常后，对应的异常服务例程(ESR)就会执行。为了决定 ESR 的入口地址， 内核使用了向量表查表机制。这里使用一张向量表，向量表其实是一个 WORD（32 位整数）数组，每个下标对应一种异常，该下标元素的值则是该 ESR 的入口地址。向量表在地址空间中的位置是可以设置的，通过 NVIC 中的一个重定位寄存器来指出向量表的地址。

​ 在复位后，该寄存器的值为 0。因此，在地址 0 （即 FLASH 地址 0） 处必须包含一张向量表，用于初始时的异常分配。要注意的是这里有个另类： 0 号类型并不是什么入口地址，而是给出了复位后 MSP 的初值。

​ 向量表从 FLASH 的 0 地址开始放置，以 4 个字节为一个单位，地址 0 存放的是栈顶地址，0X04 存放的是复位程序的地址，以此类推。从代码上看，向量表中存放的都是中断服务函数的函数名，可我们知道 C 语言中的函数名就是一个地址。

**DCD**：分配一个或者多个以字为单位的内存，以四字节对齐，并要求初始化这些内存。在向量表中， DCD 分配了一堆内存，并且以 ESR 的入口地址初始化它们。

通俗的说：内核在异常时会访问这个表地址，并根据异常类型查表，按表跳转到异常处理函数执行。

各种中断处理程序 xxx_Handler

  1. 复位程序

作用：弱符号导出本程序Reset_Handler，导入 SystemInit 和 __main 两个符号，并 LDR 数据到 R0 寄存器，跳转执行( BLX / BX )。

注：这里需要了解ARM寄存器的作用

PORC 标记了程序的开始，ENDP 标记了程序的末尾。与 R13(SP) 寄存器相关。
